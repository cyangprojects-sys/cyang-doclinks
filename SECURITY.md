# Security Policy

## Reporting a Vulnerability

If you believe you have found a security issue in cyang-doclinks or DocLinks, please report it privately:

- security@cyang.io
- legal@cyang.io

Please include:
- the affected route, workflow, or component
- reproduction steps
- expected impact
- any proof-of-concept details needed for verification

## Please Do Not

- open a public GitHub issue for a vulnerability
- run destructive tests against production systems
- attempt social engineering, credential attacks, or unauthorized data access

## What to Expect

We review reports based on severity, exploitability, and customer impact. Valid issues may lead to mitigation, patching, temporary restrictions, or broader incident handling depending on scope.

For the fuller public security posture, see:
- [docs/SECURITY_POLICY.md](docs/SECURITY_POLICY.md)
- https://www.cyang.io/security-disclosure
